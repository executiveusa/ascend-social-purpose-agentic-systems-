# Spec: Tenant-Agent-Pack

## Purpose

A tenant-agent-pack is a declarative manifest of which agents, models, tools, and workspaces a tenant gets. It is generated by missionctl and consumed by the Hermes provisioner.

## Pack schema

```json
{
  "packId": "northwest-nonprofit-standard",
  "version": "1.0.0",
  "description": "Standard agent pack for PNW nonprofits",
  "agents": [
    {
      "name": "grant-scout",
      "model": "standard",
      "tools": ["drive.read", "calendar.read"],
      "approvalPolicy": "auto-draft",
      "description": "Scans for grant opportunities matching tenant profile"
    },
    {
      "name": "campaign-drafter",
      "model": "cheap",
      "tools": ["postiz.schedule"],
      "approvalPolicy": "approval-required",
      "description": "Drafts social media campaigns from ICM stage outputs"
    },
    {
      "name": "outcome-tracker",
      "model": "cheap",
      "tools": [],
      "approvalPolicy": "auto-green",
      "description": "Tracks and summarizes program outcomes"
    }
  ],
  "litellmModels": ["openai/gpt-4.1-mini", "anthropic/claude-sonnet-4.5"],
  "openWebuiWorkspace": true,
  "langfuseProject": true
}
```

## Default packs

### northwest-nonprofit-standard
- grant-scout (standard model, read-only tools)
- campaign-drafter (cheap model, postiz.schedule with approval)
- outcome-tracker (cheap model, no external tools)

### youth-sports
- grant-scout (standard model, read-only tools)
- schedule-coordinator (cheap model, calendar.create with approval)
- safety-reviewer (critical model, no external tools, auto-draft)

### community-arts
- grant-scout (standard model, read-only tools)
- event-promoter (cheap model, postiz.schedule with approval)
- impact-storyteller (standard model, drive.read)

## Generation

```bash
missionctl agent-pack create <tenant> <pack-name>
```

Creates:
- `mission-data/<tenant>/agent-packs/<pack-name>.json`
- Updates `managed_agents` table if Hermes is provisioned

## Validation rules

- Agent names must be unique within a pack.
- Model tier must be one of: cheap, standard, critical.
- Tools must exist in `config/tool-allowlist.json`.
- Approval policy must be one of: auto-green, auto-draft, approval-required.
- `litellmModels` must be valid LiteLLM model identifiers.
