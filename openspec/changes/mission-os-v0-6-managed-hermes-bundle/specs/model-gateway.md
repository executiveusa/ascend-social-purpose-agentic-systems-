# Spec: Model Gateway

## Purpose

LiteLLM as the centralized model gateway for all agent and task model calls. Mission OS manages model config, key injection, and usage tracking.

## Architecture

```
Agent/Task → Mission OS API → LiteLLM Proxy → Model Provider
                                    ↓
                              Langfuse (trace)
                                    ↓
                              Mission OS DB (usage event)
```

## LiteLLM config

Generated by `missionctl bundle init` from `config/model-policy.json`:

```yaml
# deploy/litellm/config.yaml
model_list:
  - model_name: cheap
    litellm_params:
      model: ollama/qwen2.5:7b
      api_base: http://ollama:11434
  - model_name: standard
    litellm_params:
      model: openai/gpt-4.1-mini
      api_key: os.environ/LITELLM_API_KEY
  - model_name: critical
    litellm_params:
      model: anthropic/claude-sonnet-4.5
      api_key: os.environ/LITELLM_API_KEY

litellm_settings:
  drop_params: true
  set_verbose: false

general_settings:
  master_key: os.environ/LITELLM_MASTER_KEY
  database_url: os.environ/LITELLM_DB_URL
```

## Sync

`packages/core/src/litellm-sync.js`:
- `syncModels(tenantId)` — push model config to LiteLLM
- `pullUsage(tenantId)` — pull usage stats from LiteLLM
- `rotateKeys(tenantId)` — rotate API keys

## missionctl

```bash
missionctl litellm sync <tenant>     # Sync model config
missionctl litellm usage <tenant>    # Pull usage report
```

## Usage tracking

Every model call logs a `model.call` event with:
- `model_route` (cheap/standard/critical)
- `cost_estimate_cents`
- `input_tokens`, `output_tokens`
- `trace_id` (Langfuse trace)

## Rules

- LiteLLM master key is in Mission OS DB, injected at bundle apply time.
- Per-tenant API keys are stored in `tenants.profile.apiKeys`.
- Model calls without a valid tenant context are rejected.
- Cost estimates are written to the event journal immediately after the call.
