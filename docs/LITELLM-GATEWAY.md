# LiteLLM Model Gateway

## Purpose

LiteLLM is the centralized model gateway for all agent and task model calls in Mission OS v0.6.

## Architecture

```
Agent/Task → Mission OS API → LiteLLM Proxy → Model Provider
                                    ↓
                              Langfuse (trace)
                                    ↓
                              Mission OS DB (usage event)
```

## Configuration

Generated by `missionctl litellm sync <slug>`:

```
litellm/
  config.yaml    # Model list, virtual keys, budgets
  env            # API keys, database URL
```

## Model tiers

| Tier | Model | Use case | Cost |
|---|---|---|---|
| cheap | ollama/qwen2.5:7b | Extract, format, classify, summarize | Free (local) |
| standard | openai/gpt-4.1-mini | Draft, compare, research synthesis | ~$0.15/1M tokens |
| critical | anthropic/claude-sonnet-4.5 | Grant strategy, youth safety, legal | ~$3.00/1M tokens |

## Safety

- Master key generated at provision time, not committed
- Provider API keys are placeholders in generated config
- Per-surface budgets prevent runaway costs
- All calls logged to Langfuse
