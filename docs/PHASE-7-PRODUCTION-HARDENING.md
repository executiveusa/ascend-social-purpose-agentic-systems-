## Acceptable dry-run limitations

These limitations are acceptable for Phase 7 (pre-production demo mode):

1. **No live containers** — All deployment commands are `--dry-run`. Bundle files are generated locally.
2. **File-backed state** — `mission-data/` is local JSON. Real Postgres for multi-tenant production is Phase 8.
3. **No real agent execution** — Hermes/Pi/Absurd/Sandcastle adapter seams exist but don't call live services.
4. **No live model routing** — LiteLLM config is generated; actual model calls require API keys.
5. **No live Langfuse traces** — Trace metadata is generated; actual Langfuse sync needs running instance.

## What requires real VPS/external credentials later

- `POSTGRES_PASSWORD` and `DATABASE_URL` — set in `.env.managed` (gitignored, generated by `missionctl bundle up`)
- `LITELLM_OPENAI_API_KEY`, `LITELLM_ANTHROPIC_API_KEY` — set post-deployment
- `HERMES_IMAGE` — pin to known-good Hermes image before production
- `LANGFUSE_PUBLIC_KEY`, `LANGFUSE_SECRET_KEY` — set after `missionctl langfuse sync`
- Hostinger VPS IP — set in `missionctl hostinger handoff --vps-ip`
