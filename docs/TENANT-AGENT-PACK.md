# Tenant-Agent-Pack

## Purpose

A tenant-agent-pack is a deterministic manifest of agents, skills, tools, schedules, policies, and prompts for a specific tenant. It is generated by `missionctl pack generate` and consumed by the Hermes provisioner.

## Structure

```
tenant-agent-pack/
  manifest.yaml                    # Pack metadata, agents, models
  org/
    org-profile.json               # Tenant profile (name, mission, programs)
    brand.json                     # Brand colors, tone
    locales/en-US.json             # Localization
  hermes/
    config.patch.yaml              # Hermes profile config
    SOUL.md                        # Agent identity
    MEMORY.md                      # Runtime state
    USER.md                        # Staff context
    skills/                        # 8 skill definitions
      founder.md
      grants.md
      comms.md
      programs.md
      board-packet.md
      donor-followup.md
      volunteer-coordinator.md
      impact-report.md
    schedules/                     # 3 cron schedules
      opportunity-radar.cron.yaml
      weekly-board-digest.cron.yaml
      monthly-impact-report.cron.yaml
    tools/
      allowlist.yaml               # Tool policy (from config/tool-allowlist.json)
      env-map.yaml                 # Environment variable mapping
  mission/
    policy.yaml                    # Approval classes and hard blocks
    workflows.yaml                 # Available workflows
    approvals.yaml                 # Auto-approve rules
    dashboard-panels.yaml          # Dashboard panel config
  openwebui/
    models.json                    # Available models
    workspace.json                 # Workspace config
    starter-agents.json            # 6 starter assistants
  prompts/
    board-packet.md
    donor-followup.md
    grant-triage.md
    impact-story.md
  tests/
    smoke.json
    evals.json
```

## Commands

```bash
missionctl pack generate <slug>    # Create pack (33 files)
missionctl pack validate <slug>    # Verify all 33 files exist
missionctl pack publish <slug>     # Log publish event
```

## Generation rules

- Pack version is timestamped: `1.0.0-<unix_ms>`
- Skill files are templated from `missionctl/templates/hermes/skills/`
- Tool allowlist is read from `config/tool-allowlist.json`
- Generation is deterministic (same tenant → same structure)
- Pack is stored under `mission-data/<tenant>/tenant-agent-pack/`
