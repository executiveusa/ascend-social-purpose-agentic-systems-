## Rust services gap

All 5 Rust crates are scaffolds (`main.rs` only, no business logic):

| Crate | Intended purpose | v0.6 scope |
|---|---|---|
| mission-core-rs | Authoritative tenant/CRM/approval core | Defer to v0.7 — JS core is sufficient for v0.6. |
| mission-connect-rs | Public frontend bridge | Defer — JS bridge works. |
| mission-worker-rs | Outbox worker + scheduled jobs | Defer — JS outbox works. |
| mission-policy-rs | Deterministic action-risk classifier | Defer — JS safety.js works. |
| mission-icm-rs | Filesystem-safe ICM stage runner | Defer — JS icm.js works. |

**Rationale:** v0.6 is a productization pass, not a Rust migration. The JS layer
is working and tested. Rust migration is a v0.7+ concern.
