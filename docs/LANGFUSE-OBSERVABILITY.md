# Langfuse Observability

## Purpose

Self-hosted Langfuse provides trace-level observability for all agent runs, model calls, and workflow execution in Mission OS v0.6.

## Architecture

```
Mission OS API ──── trace_id ────→ Langfuse
   │                                  │
   ├── agent.run event                ├── trace detail
   ├── model.call event               ├── model usage
   ├── external.action event          ├── tool call
   └── approval.* events              └── approval context
```

## Configuration

Generated by `missionctl langfuse sync <slug>`:

```
langfuse/
  env                    # Langfuse environment
  trace-metadata.json    # Trace tag schema and redaction policy
```

## Trace metadata tags

Every trace carries these tags:
- `org_id`
- `tenant_id`
- `surface` (mission-os, openwebui, hermes-*)
- `workflow_kind`
- `approval_class` (green/yellow/orange/red)
- `agent_slug`
- `run_id`
- `artifact_id`
- `release_id`
- `environment`

## Redaction

- No PII in trace metadata for restricted workflows
- Redaction policy: "no PII in trace metadata for restricted workflows"
- Youth/family/donor data is never logged in traces

## Self-hosted

Langfuse runs as a Docker service in the managed bundle, using the shared Postgres with a separate schema. No external API dependency.

## Retention

- Traces retained for 90 days (configurable)
- Purged by scheduled job in outbox worker
