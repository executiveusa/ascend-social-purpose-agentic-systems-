# Mission OS v0.6 — Managed Hermes Bundle

## Overview

v0.6 extends Mission OS from a deployment handoff system (v0.5) to a managed agent runtime bundle. It adds Hermes as a managed execution runtime, tenant-agent-packs, LiteLLM/Open WebUI/Langfuse bundle templates, and extends missionctl into a one-click flywheel.

## What's new in v0.6

### missionctl commands (dry-run capable)

```bash
# Bundle management
missionctl bundle up <slug> [--dry-run]      # Generate managed bundle
missionctl bundle status <slug>               # Check bundle status
missionctl bundle smoke <slug> [--dry-run]    # Verify bundle integrity
missionctl bundle release <slug>              # Mark bundle as released
missionctl bundle down <slug>                 # Stub for teardown

# Tenant-agent-pack
missionctl pack generate <slug>               # Create agent pack
missionctl pack validate <slug>               # Verify pack completeness
missionctl pack publish <slug>                # Publish pack

# Hermes runtime
missionctl hermes provision <slug>            # Provision Hermes config
missionctl hermes health <slug>               # Check Hermes health

# Service sync
missionctl litellm sync <slug>                # Generate LiteLLM config
missionctl langfuse sync <slug>               # Generate Langfuse config
missionctl openwebui sync <slug>              # Generate Open WebUI config
```

### Architecture

```
missionctl (one-click flywheel)
     |
     +-- bundle up → handoff/<tenant>/managed/
     |       |-- docker-compose.managed.yml (postgres, redis, mission-api, site, caddy, hermes, litellm, langfuse, open-webui, prometheus, grafana)
     |       |-- .env.managed
     |       |-- Caddyfile.managed
     |       |-- hermes/ (env, SOUL.md, MEMORY.md, USER.md, skills/)
     |       |-- litellm/ (config.yaml, env)
     |       |-- open-webui/ (env, workspace.json, starter-agents.json)
     |       |-- langfuse/ (env, trace-metadata.json)
     |       |-- prometheus.yml
     |       |-- grafana-dashboard.json
     |       |-- smoke-test.managed.sh
     |       |-- release-manifest.json
     |
     +-- pack generate → mission-data/<tenant>/tenant-agent-pack/
             |-- manifest.yaml
             |-- org/ (profile, brand, locales)
             |-- hermes/ (config, SOUL, MEMORY, USER, skills, schedules, tools)
             |-- mission/ (policy, workflows, approvals, dashboard-panels)
             |-- openwebui/ (models, workspace, starter-agents)
             |-- prompts/ (board-packet, donor-followup, grant-triage, impact-story)
             |-- tests/ (smoke, evals)
```

### Safety

- Hermes dashboard bound to 127.0.0.1:8765 — NOT public
- Grafana bound to 127.0.0.1:3002 — NOT public
- Prometheus bound to 127.0.0.1:9090 — NOT public
- Hermes browser/shell tools disabled by default
- Approval policy enforced (green=auto, yellow=draft, orange=approval, red=restricted)
- No automatic grant submission, legal filing, or youth/donor communication

### What works now (dry-run)

- All missionctl v0.6 commands execute successfully in dry-run mode
- Tenant-agent-pack generation is deterministic (33 files)
- Bundle generation creates complete managed Docker Compose with all services
- Smoke test verifies 12 checks including Hermes-not-public safety
- Release manifest tracks git SHA, pack version, service status

### What is still stubbed/dry-run

- No live Docker deployment (dry-run only)
- No real Hermes container running
- No real LiteLLM API calls
- No real Langfuse traces
- No real Open WebUI workspace
- Provider API keys are placeholders
- DB/RBAC/event journal not yet implemented (P1 scope)

## Demo tenant

missionctl hermes provision demo-pnw
missionctl litellm sync demo-pnw
missionctl langfuse sync demo-pnw
missionctl openwebui sync demo-pnw
missionctl bundle up demo-pnw --dry-run
missionctl bundle smoke demo-pnw --dry-run

All commands pass ✅
