# Open WebUI Workspace

## Purpose

Open WebUI is the staff-facing AI workspace. It provides a familiar ChatGPT-style interface for nonprofit staff to interact with AI assistants.

## Architecture

- Open WebUI connects ONLY to LiteLLM (never directly to model providers)
- Each tenant gets workspace config, starter agents, and model list
- Open WebUI is NOT the system of record — Mission OS is
- Open WebUI does NOT bypass Mission OS approvals

## Configuration

Generated by `missionctl openwebui sync <slug>`:

```
open-webui/
  env                    # Environment (LiteLLM endpoint, auth, branding)
  workspace.json         # Workspace config (models, settings)
  starter-agents.json    # 6 starter assistants
```

## Starter assistants

1. **Mission Assistant** — General-purpose daily operations
2. **Grant Drafting Assistant** — Grant applications and opportunity analysis
3. **Donor Follow-Up Assistant** — Donor communications (approval-gated)
4. **Volunteer Coordinator Assistant** — Volunteer matching and scheduling
5. **Board Packet Assistant** — Board meeting packets
6. **Impact Story Assistant** — Impact narratives from outcome data

## Model access

- `cheap` — Quick Assistant (summaries, drafts)
- `standard` — Standard Assistant (most tasks)
- `critical` — Critical Assistant (grants, legal, strategic)

## Safety

- Signup disabled (auth managed by Mission OS)
- No provider master keys exposed
- LiteLLM virtual key is scoped to openwebui surface only
- Monthly budget limit enforced by LiteLLM
